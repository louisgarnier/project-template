# Workflow — How the AI Behaves

> Enforced via `.windsurfrules` at project root.
> These files tell the AI HOW to work. Requirements (WHAT to build) are in `docs/project/requirements/`.

| File | Trigger | Purpose |
|------|---------|---------|
| `GIT_WORKFLOW.md` | Committing code | Commit convention, push rules |
| `ADR.md` | Technical decision | Log decisions, check past decisions |
| `ERRORS.md` | Debugging error | Check known errors, investigation method |
| `LOGGING.md` | Adding logs | Track what's logged, avoid duplicates |
